# RL-LLM Tutor

An offline reinforcement learning system that trains an intelligent tutoring agent to teach 6th-grade math. The agent learns to select the best pedagogical action at each turn of a tutoring dialogue, powered by a Mistral LLM for both the student and tutor simulations, and trained using offline RL algorithms (BC, DQN, CQL) via [d3rlpy](https://github.com/takuseno/d3rlpy).

## Overview

The RL-LLM Tutor frames math tutoring as a **Markov Decision Process (MDP)**:

- **State**: A 25-dimensional vector encoding the current tutoring dialogue (18-dim sentence embedding via `all-MiniLM-L6-v2` + 7 handcrafted features).
- **Actions (4)**: `Instruct`, `Encourage`, `Refocus`, `Question` — pedagogical moves a tutor can make.
- **Reward**: `+1` if the student solves the problem, `-1` on timeout, `0` otherwise (with optional shaping for early success).
- **Environment**: Dialogue episodes of up to 8 turns, where both the tutor and student responses are generated by a Mistral LLM.

The pipeline collects offline interaction data, optionally augments it via a fitted Q-function, then trains and compares multiple offline RL policies.

## How It Works

### 1. Environment (`environment.py`)

`TutorEnv` is a Gymnasium environment that simulates a tutoring session. On each step:
1. The **tutor** calls the Mistral API with an action-specific system prompt (e.g. "give a hint", "encourage", "ask a question").
2. The **student** responds, simulated as a struggling 6th-grader who only gives a numerical answer when confident.
3. The episode ends when the student gives the correct answer (`reward = +1`) or the turn limit is reached (`reward = -1`).

### 2. State Encoding (`state_extractor.py`)

Each dialogue is encoded into a 25-dimensional state vector:
- **Dims 0–17**: First 18 components of a `all-MiniLM-L6-v2` sentence embedding of the full dialogue.
- **Dims 18–24**: 7 binary/scalar features: math keyword presence, solved signal, re-explanation request, tutor-student repetition, off-topic signal, unrelatedness, student question mark.

### 3. Data Generation (`data_generator.py`)

Generates offline datasets by running `N_DIALOGUES` (default 2,000) episodes in parallel using a thread pool. Each episode uses a random action policy. Results are checkpointed every `CKPT_EVERY` episodes and saved as `.npy` arrays.

### 4. Augmentation (`augmentation.py`)

Improves dataset quality using a **Q-function-guided augmentation** strategy:
1. Fit a `ExtraTreesRegressor` Q-function via Fitted Q-Iteration (50 iterations).
2. Identify the top-N transitions where switching to the Q-optimal action yields the highest gain.
3. Re-run those episodes with the improved action forced at turn 0.
4. Concatenate augmented transitions with the original dataset.

Two augmentation rounds are supported (`_aug` and `_aug2`).

### 5. Training (`trainer.py`)

Trains 9 models via d3rlpy:

| Model | Algorithm | Dataset |
|---|---|---|
| `bc` | Behaviour Cloning | D (baseline) |
| `q` | DQN | D (baseline) |
| `cql` | Discrete CQL | D (baseline) |
| `bc_aug` | Behaviour Cloning | D+ (augmented) |
| `q_aug` | DQN | D+ (augmented) |
| `cql_aug` | Discrete CQL | D+ (augmented) |
| `super_cql_s7` | Discrete CQL (α=8) | D++ (seed 7) |
| `super_cql_s13` | Discrete CQL (α=8) | D++ (seed 13) |
| `super_cql_s42` | Discrete CQL (α=8) | D++ (seed 42) |

Each model trains for 150,000 steps (15 epochs × 10,000 steps/epoch), batch size 512.

### 6. Evaluation (`evaluator.py`)

Policies are evaluated over live episodes in `TutorEnv` using `N_EVAL` (default 15) episodes. Success rate and 95% confidence intervals are reported. The `SuperCQL` policy uses **majority-vote ensembling** across the 3 seeds.

## Algorithms

### Behaviour Cloning (BC)
Supervised imitation of the offline dataset. Learns to mimic average behaviour — a simple but strong baseline.

### DQN (Offline)
Standard Q-learning on the offline dataset. Can overestimate Q-values for out-of-distribution actions.

### Conservative Q-Learning (CQL)
Penalizes Q-values for out-of-distribution actions, making it more robust to distributional shift in offline settings. `alpha` controls the conservatism penalty.

### SuperCQL
CQL with a higher conservatism penalty (`alpha=8.0`, `lr=3e-5`) trained on the twice-augmented dataset (`D++`). An ensemble of 3 models (seeds 7, 13, 42) votes on the final action.

## Setup & Installation

### Prerequisites

- Python 3.9+
- CUDA GPU recommended (code auto-detects `cuda:0` or falls back to CPU)
- A [Mistral AI](https://mistral.ai) API key

### Install dependencies

```bash
pip install d3rlpy==2.4.0 \
            gymnasium==0.29.1 \
            sentence-transformers==2.7.0 \
            datasets==2.20.0 \
            scikit-learn>=1.3 \
            scipy>=1.11 \
            numpy>=1.24 \
            pandas>=2.0 \
            matplotlib>=3.7 \
            seaborn>=0.12 \
            tqdm requests
```

### Set your API key

```bash
export MISTRAL_API_KEY="your_key_here"
```

> **Note:** If `MISTRAL_API_KEY` is not set, `llm_utils.py` falls back to mock responses for offline testing.

## Usage

### Run the full pipeline (Kaggle notebook)

Open `rl-final-notebook-kaggle.ipynb` in a Kaggle environment with 2×T4 GPUs. The notebook runs all steps end-to-end:

1. Install → Setup directories → Write module files
2. Verify API key
3. Generate baseline dataset D (2,000 episodes)
4. Augment to D+ (3,000 transitions)
5. Augment to D++ (3,500 transitions)
6. Train all 9 models
7. Evaluate all policies
8. Generate plots

### Run individual modules

```python
import sys
sys.path.insert(0, "src")

from config import *
from data_generator import generate_dataset
from augmentation import augment
from trainer import train_all
from evaluator import eval_all
import json

# Load problems
problems = json.load(open("data/math_problems.json"))

# Generate baseline dataset
generate_dataset(problems, N_DIALOGUES, DATA_DIR, n_workers=N_WORKERS)

# Train all models
trained = train_all(DATA_DIR, MODELS_DIR)

# Evaluate
results = eval_all(trained, problems, n_eval=N_EVAL)
```

## Configuration

All hyperparameters are defined in `src/config.py`:

| Parameter | Default | Description |
|---|---|---|
| `MISTRAL_MODEL` | `mistral-small-latest` | Mistral model to use |
| `MAX_TURNS` | `8` | Maximum turns per episode |
| `N_DIALOGUES` | `2000` | Episodes for baseline dataset |
| `N_AUG_TARGET` | `3000` | Target transitions after 1st augmentation |
| `N_AUG2_SAMPLES` | `500` | Additional transitions from 2nd augmentation |
| `N_EVAL` | `15` | Episodes per policy evaluation |
| `DISCOUNT` | `0.9` | RL discount factor (γ) |
| `STATE_DIM` | `25` | State vector dimension |
| `N_ACTIONS` | `4` | Number of tutoring actions |
| `BATCH_SIZE` | `512` | Training batch size |
| `CQL_ALPHA` | `4.0` | CQL conservatism penalty |
| `SUPER_CQL_ALPHA` | `8.0` | SuperCQL conservatism penalty |
| `N_TRAIN_STEPS` | `150,000` | Training steps per model |
| `REWARD_SHAPE_BONUS` | `0.5` | Bonus for early problem solving |
| `SUPER_SEEDS` | `[42, 7, 13]` | Random seeds for ensemble |

## Data & Augmentation

The `data/` directory contains three dataset variants:

| Suffix | Description | Approx. Transitions |
|---|---|---|
| *(none)* | Baseline random-policy dataset | ~2,000 × avg_turns |
| `_aug` | After 1st Q-guided augmentation | ~3,000 × avg_turns |
| `_aug2` | After 2nd Q-guided augmentation | ~3,500 × avg_turns |
| `*_raw` | Raw episodes before merging with base | — |

Each dataset is saved as 4 `.npy` files: `states`, `actions`, `rewards`, `terminals`.

## Evaluation

Evaluation runs live episodes using the trained policy in `TutorEnv`. Results are reported as:

```
success=73.33% +/-22.40%
```

Policies are ranked by mean success rate. The `SuperCQL` ensemble uses majority-vote across 3 independently-seeded models.
